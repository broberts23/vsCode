# Wiki Formatting Guide

This document defines the standard format for Azure DevOps Wiki entries generated by the Documentation Copilot.

## Page Structure

Every generated wiki page follows this template:

```markdown
# Module: {ModuleName}

## Overview
{One-paragraph description of the module's purpose.}

## Module Path
`{file_path}`

## Dependencies
**Internal Dependencies:**
- `package.module`

**External Dependencies:**
- `third_party_lib`

## Functions
### `function_name()`
{Description of the function.}

- **Line:** 42
- **Decorators:** `@staticmethod`
- **Return Type:** `bool`

**Parameters:**
| Parameter | Type | Description |
| --- | --- | --- |
| `param1` | `str` | The input string. |

## Classes
### `ClassName`
{Description of the class.}

- **Line:** 120
- **Base Classes:** `BaseClass`

**Methods:**
| Method | Parameters | Return Type |
| --- | --- | --- |
| `method_name` | `self, arg: str` | `None` |

## Workflow Diagrams
### Class Diagram
::: mermaid
classDiagram
    class ClassName {
        +method_name(self, arg: str) None
    }
    BaseClass <|-- ClassName
:::
```

## Markdown Compatibility

Azure DevOps Wiki supports:

- Standard Markdown (headings, lists, tables, code blocks)
- Mermaid diagrams using `::: mermaid` fence
- Backtick inline code
- HTML is NOT supported inside Mermaid blocks
- The `flowchart` Mermaid keyword is NOT supported — use `graph` instead
