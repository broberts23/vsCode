# Wiki Documentation Schema

Defines the standard schema for wiki entries generated by the Documentation Copilot.

## Entry Structure

```typescript
interface WikiEntry {
  title: string;                    // "Module: {ModuleName}"
  sections: WikiSection[];
}

interface WikiSection {
  heading: string;                  // H2 heading text
  body: string;                     // Markdown body content
}
```

## Standard Sections

| Section | Heading | Condition |
|---|---|---|
| 1 | Overview | Always |
| 2 | Module Path | Always |
| 3 | Dependencies | If module has imports |
| 4 | Functions | If module has 1+ functions |
| 5 | Classes | If module has 1+ classes |
| 6 | Workflow Diagrams | If module has 3+ functions or 1+ classes |

## Function Entry Format

```
### `function_name()`
{docstring or "No docstring provided."}

- **Line:** 42
- **Decorators:** `@staticmethod`, `@route`
- **Return Type:** `bool`

**Parameters:**
| Parameter | Type | Description |
| --- | --- | --- |
| `param1` | `str` | Description. |
```

## Class Entry Format

```
### `ClassName`
{docstring or "No docstring provided."}

- **Line:** 120
- **Base Classes:** `BaseClass`, `Mixin`

**Methods:**
| Method | Parameters | Return Type |
| --- | --- | --- |
| `method_name` | `self, arg: str` | `None` |
```

## Diagram Inclusion Rules

| Condition | Diagram Type |
|---|---|
| 1 class | No diagram |
| 2+ classes | `classDiagram` with inheritance |
| 3+ functions | `sequenceDiagram` if decorators indicate roles, else `graph TD` |

## Page Path Convention

```
API-Reference/{TargetName}/{ModuleName}
```

Examples:

- `API-Reference/calculate_total/calculator`
- `API-Reference/AuthService/auth`
- `API-Reference/parse_config/config_loader`
